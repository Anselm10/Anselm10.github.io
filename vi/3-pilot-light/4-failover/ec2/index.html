EC2

Khởi chạy EC2

1.1 Nhấp vào AMIs để điều hướng đến bảng điều khiển ở khu vực Oregon (us-west-2).

Bạn sẽ cần đợi cho đến khi pilotAMI có trạng thái là Available trước khi tiếp tục bước tiếp theo. Điều này có thể mất vài phút.

1.2 Chọn pilotAMI. Nhấp vào nút Launch instance from AMI.

1.3 Nhập pilot-secondary vào trường Name.

1.4 Trong phần Key pair (login), chọn Proceed without a key pair (not recommended) làm Key pair name.

1.5 Trong phần Network settings, nhấp vào nút Edit.

1.6 Chọn VPC không phải là VPC mặc định. Chọn Select existing security group cho Firewall (security groups) và sau đó chọn network-stack-WebServerSecurityGroup-xxx làm Common security groups.

1.7 Mở rộng phần Advanced details. Chọn pilot-secondary-S3InstanceProfile-xxx làm IAM instance profile.

1.8 Bạn sẽ muốn cấu hình bootstrap cho phiên bản để có các cấu hình cần thiết để ứng dụng Unishop ở khu vực Oregon (us-west-2) có thể kết nối với cơ sở dữ liệu primary writer mới ở khu vực Oregon (us-west-2). Bạn sử dụng user data để thực hiện điều này.

Sao chép và dán script dưới đây vào trường User data, sau đó nhấp vào nút Launch instance.

User Data Script:

#!/bin/bash     
sudo su ec2-user                        
export AWS_DEFAULT_REGION=$(curl -s http://169.254.169.254/latest/dynamic/instance-identity/document | python -c "import json,sys; print json.loads(sys.stdin.read())['region']")
export DATABASE=$(aws rds describe-db-clusters --region $AWS_DEFAULT_REGION --db-cluster-identifier pilot-secondary --query 'DBClusters[*].[Endpoint]' --output text)
sudo bash -c "cat >/home/ec2-user/unishopcfg.sh" <<EOF
#!/bin/bash
export DB_ENDPOINT=$DATABASE
EOF
sudo systemctl restart unishop

Bạn sẽ cần đợi cho đến khi phiên bản được khởi chạy thành công. Bạn có thể nhấp vào nút View all instances để điều hướng đến bảng điều khiển EC2. Xác minh rằng pilot-secondary có trạng thái Running. Điều này có thể mất vài phút.
